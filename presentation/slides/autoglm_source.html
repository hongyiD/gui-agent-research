Open-AutoGLM 源码结构

模块化设计，清晰分离核心逻辑、模型交互与底层操作

main.py Entry

项目主入口 CLI。负责解析参数、环境检查、初始化并启动 Agent。

phone_agent/agent.py Core

核心 PhoneAgent 类。管理状态循环，协调感知、思考与行动。

phone_agent/model/

模型交互层。封装 API 请求逻辑 (Client) 与 Prompt 构建 (MessageBuilder)。

phone_agent/actions/

动作处理层。解析模型输出的 JSON，分发具体指令。

phone_agent/adb/

底层驱动层。封装 ADB 命令，处理截图、点击、滑动等物理操作。
